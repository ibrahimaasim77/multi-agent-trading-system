# Trades — Daily Journal

One file per trading day: `YYYY-MM-DD.md`. Auto-generated by the post-mortem agent.

## Format

Each file has a YAML frontmatter block (machine-parseable) followed by Markdown prose (human-readable). Both matter — frontmatter feeds metrics, prose feeds next-day reasoning.

### Schema

```yaml
---
date: YYYY-MM-DD              # ISO date, always today
market_open: true             # false on holidays/weekends

trades_placed:                # zero or more
  - routine: morning          # morning | intraday
    ticker: QQQ
    side: buy                 # always buy in current system
    dollar_amount: 100        # what we deployed
    limit_price: 697.50       # the price we entered at
    fill_price: 697.45        # what we actually got
    fill_time: "09:31 ET"
    close_price: 691.20       # session close for the ticker
    return_pct: -0.89         # (close - fill) / fill * 100
    outcome: loss             # win | loss | scratch | unfilled

stand_aside_candidates:       # tickers an agent considered but skipped
  - ticker: AMD
    routine: morning
    reasoning: "unconfirmed Iran rebound, abort trigger"
    close_change_pct: 3.1
    correctness: missed       # missed | correct

agents_aborted: false         # true if any routine hit a hard abort

daily_score: 50               # 0-100, agent's self-assessment
account_value_close: 217.19
cash_close: 217.19
---
```

### Prose sections

1. **Day summary** — what was today's macro story; did the agents read it correctly?
2. **Trade-by-trade review** — for each placed trade, the thesis and the outcome
3. **Stand-aside review** — for each skipped candidate, was that the right call?
4. **Patterns noted** — references to prior journal entries; recurring lessons
5. **Recommendations for tomorrow** — concrete, specific changes to consider

## Why this format

- **YAML frontmatter** is parseable for the metrics aggregator without complex extraction
- **Markdown prose** is what the morning agent reads to learn from yesterday — natural language carries nuance that JSON would strip
- **One file per day** means git diffs are clean and the file list IS the trading history
- **No editing of old entries** — past decisions are immutable. New learnings update tomorrow's prompt, not yesterday's record.

## Reading the journal

The morning routine reads the most recent ~14 entries before deciding. The post-mortem agent reads all entries when looking for patterns.

A human reader: open any file to see the full reasoning for that day.

## Anti-patterns

- Do not edit old journal entries to make them look better. Honesty is the input to learning.
- Do not store credentials or private notes in these files. The repo is public.
- Do not skip writing the journal on stand-aside days. Stand-aside decisions are also data.
